## 상세

### 1. OpenClaw — 개인 에이전트의 표준 🦞
⭐374k+로 GitHub 역사상 최속 성장 오픈소스. "개인 에이전트는 당신이 이미 있는 곳에 살아야 한다"는 철학으로 WhatsApp·Telegram·Signal·Discord·iMessage에서 구동. Personal Stack의 '프론트엔드'를 메신저로 정의한 프로젝트.
- 출처: [ByteByteGo](https://blog.bytebytego.com/p/top-ai-github-repositories-in-2026), [aimultiple](https://aimultiple.com/open-source-web-agents)

### 2. Superpowers 스킬 프레임워크 ⚡
⭐243k+(7/1 기준)로 커뮤니티 스킬 프레임워크 중 압도적 1위. Claude Code에 스킬 묶음을 체계적으로 설치·관리하는 사실상의 표준 레이어.
- 출처: [Developers Digest](https://www.developersdigest.tech/blog/best-claude-code-skills-2026)

### 3. n8n 에이전틱 워크플로 🔄
⭐196.4k. 2026년의 전환점은 프롬프트 체이닝 → 자율 에이전트(추론·자가수정·핸드오프). 자연어로 워크플로를 만들어주는 n8n AI Assistant 내장. Claude·Gemini·OpenAI를 오케스트레이션하는 개인 자동화 허브로 쓰임.
- 출처: [n8n Blog](https://blog.n8n.io/we-need-re-learn-what-ai-agent-development-tools-are-in-2026/), [Entrans](https://www.entrans.ai/blog/n8n-workflow-automation-trends)

### 4. Karpathy 행동원칙 CLAUDE.md 📜
Karpathy가 올린 "AI 코딩 에이전트의 답답한 점"을 개발자 Forrest Chang이 4가지 행동 원칙의 단일 CLAUDE.md로 정리 → 수 주 만에 ⭐144k, 역대 최속 성장 AI 워크플로 저장소. 파일 하나로 에이전트 품질을 끌어올리는 최소비용 스택 개선.
- 출처: [Developers Digest](https://www.developersdigest.tech/blog/best-claude-code-skills-2026)

### 5. Context Engineering 🧠
"프롬프트 엔지니어링은 끝났다" — 구조화된 컨텍스트, 외부 지식, 재사용 스킬, 통제된 도구 접근을 설계하는 방법론으로 전환. Gartner가 AI 앱 개발의 핵심 시프트로 지목. Personal Stack 설계의 이론적 토대.
- 출처: [WeBuild-AI](https://www.webuild-ai.com/insights/aligning-spec-driven-development-and-context-engineering-for-2026), [Medium — Dave Patten](https://medium.com/@dave-patten/the-new-engineering-stack-specs-context-and-agents-f3454529768e)

### 6. Spec-Driven Development (SDD) 📋
"The spec is the prompt." DeepLearning.AI 전용 강좌 개설로 주류화. Spec Kit 사용 팀은 '처음부터 재생성' 사이클이 약 1/10로 감소, AWS Kiro는 40시간짜리 기능을 스펙 우선 작성으로 8시간 내 출시한 사례 문서화.
- 출처: [zeroshot](https://zeroshot.ghost.io/spec-driven-development-with-ai-coding-agents/), [BCMS](https://thebcms.com/blog/spec-driven-development)

### 7. Open WebUI — 셀프호스팅 허브 🏠
⭐124k+, 다운로드 2.8억. 완전 오프라인 구동 가능한 자체 호스팅 AI 플랫폼. Ollama와 페어링해 '프라이빗 ChatGPT'를 만드는 로컬 스택의 UI 표준.
- 출처: [ByteByteGo](https://blog.bytebytego.com/p/top-ai-github-repositories-in-2026)

### 8. Browser-Use — 브라우저 에이전트 1위 🌐
⭐58k+, GitHub 트렌딩 1위 경험. Odysseys 벤치마크 87.4%로 OpenAI·Anthropic·Google·Microsoft의 컴퓨터 사용 에이전트를 모두 제침. 7/1 데스크톱 앱 출시로 비개발자 접근성 확대.
- 출처: [browser-use](https://github.com/browser-use/browser-use), [Browser Use 벤치마크](https://browser-use.com/posts/ai-browser-agent-benchmark)

### 9. MCP 필수 트리오: Context7 · GitHub · Playwright 🔌
"Context7(문서 환각 제거) + GitHub MCP(레포·CI) + Playwright MCP(브라우저 자동화)가 Claude Code 워크플로의 80%를 커버"가 커뮤니티 정설로 자리잡음. 검색은 Exa가 에이전트용 검색엔진 1위.
- 출처: [Totalum](https://www.totalum.app/blog/best-mcp-servers-2026), [Builder.io](https://www.builder.io/blog/best-mcp-servers-2026)

### 10. claude-skills 337종 라이브러리 📚
Claude Code·Cursor용 스킬/플러그인/에이전트 337종 모음. 주목할 점은 마케팅·PM·경영 컨설팅 등 비개발 직군 커버 — '코딩 어시스턴트 → 업무 전반 어시스턴트'로의 진화 신호.
- 출처: [AIToolly](https://aitoolly.com/ai-news/article/2026-07-07-claude-skills-repository-launches-with-337-specialized-ai-agent-skills-for-engineering-and-business)

### 11. Anthropic frontend-design 스킬 🎨

- 출처: [Firecrawl](https://www.firecrawl.dev/blog/best-claude-code-skills)

### 12. AGENTS.md 표준 📄
에이전트에게 프로젝트 컨텍스트를 주는 사실상의 오픈 표준으로 정착. BB-Skills 등 이를 확장하는 생태계 등장. 도구 종속 없이 스택을 이식 가능하게 만드는 핵심.
- 출처: [WeBuild-AI](https://www.webuild-ai.com/insights/aligning-spec-driven-development-and-context-engineering-for-2026)

### 13. Codex 플러그인 — 스킬+MCP 번들 📦

- 출처: [Totalum](https://www.totalum.app/blog/best-mcp-servers-2026)

### 14. 로컬 오픈웨이트 스택: GLM-5.2 + Ollama 💻
GLM-5.2가 LiveBench Agentic 73.33으로 모든 프로프라이어터리 모델을 추월(오픈웨이트 최초). 런타임은 Ollama가 사실상 승자, UI는 LM Studio·Open WebUI. 코딩 에이전트용은 Kimi K2.7 Code, 노트북용은 Gemma 4 12B가 정석.
- 출처: [Pinggy](https://pinggy.io/blog/best_open_source_self_hosted_llms_for_coding/), [TechFuel HQ](https://techfuelhq.com/articles/self-hosted-ai-stack-2026/)

BrowserOS는 AI 에이전트가 내장된 '내가 모는 브라우저', Open Browser Use는 MV3 확장+네이티브 호스트로 Claude Code·Codex·CI가 실제 Chrome 탭을 조작하게 하는 로컬 우선 도구. 단일 런타임 종속 없는 브라우저 자동화 계층.
- 출처: [BrowserOS](https://github.com/browseros-ai/BrowserOS), [Product Hunt](https://www.producthunt.com/products/open-browser-use)
